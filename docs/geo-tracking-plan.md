## 一、追踪原则

### 1.1 站点特性决定追踪方式

ai-daily-brief 是 GitHub Pages 静态站点，与传统商业站点的差异：

| 维度 | 商业站点 | ai-daily-brief (纯证据容器) |
|------|---------|--------------------------|
| 转化目标 | 注册/购买/咨询 | 知识库被AI引用 |
| 用户行为 | 浏览→填表单→转化 | AI抓取→索引→生成回答时引用 |
| 追踪粒度 | 页面级（GA/埋点） | 事实级（原子事实被引用率） |
| 直接效果 | 线索/销售 | 知识库条目出现在AI回答中 |
| 间接效果 | 品牌搜索量 | 站点头部文章被搜索收录排名 |

### 1.2 三层追踪映射

| 三层诱惑链 | GEO追踪指标 | 观测方法 |
|-----------|------------|---------|
| 捕获层 (Selection) | AI搜索命中率 | 五平台搜索看是否出现在结果中 |
| 记忆层 (Retention) | 原子事实引用准确率 | 检查AI回答中引用数字是否与原文一致 |
| 传播层 (Propagation) | 跨平台引用扩散 | 同一事实在不同平台回答中出现次数 |

### 2.2 事实引用质量（每日采样）

关键指标：
- **引用召回率** = 出现在回答中的事实数 ÷ 知识库总事实数（抽样）
- **引用准确率** = 数值准确的事实数 ÷ 被引用的事实数
- **幻觉率** = AI自己编造的事实数 ÷ 回答中总事实数
