## 一、核心逻辑

三层诱惑链 = 从「半眼看世界」策划方案推导 + 适配GEO论文的双阶段框架（Selection + Absorption）

### 框架对照

| GEO论文阶段 | 三层诱惑链 | 目标 | 底层逻辑 |
|:---|:---|:---|:---|
| Citation Selection | 第一层·诱AI读 | AI检索引擎选中你的内容 | 关键词+权威+PAWC前置+AI可爬性 |
| Citation Absorption | 第二层·诱AI记住 | AI深度吸收到回答正文 | 证据密度+语义对齐+结构清晰度+引语 |
| 扩散传播 | 第三层·诱AI传播 | AI跨上下文持续引用 | 独立结论+时效性+出处声明 |

## 二、第一层·诱AI读（Selection触发）

### 目标
让AI搜索引擎在回答问题时，选中你的内容作为引用来源。

### 关键数据
- 官方+新闻+垂直三类来源占全部引用的**79%-88%**（论文数据）
- 高权威域名（DR 520-592）更容易入选
- **PAWC核心：句1权重≈句20的5倍**（Princeton论文）

### 实操方法

1. **首150字直接回答问题（PAWC前置）** — 第一句必须包含：具体数字/单位 + 核心结论概要。不得用"In today's..."等填充句开头。
2. **关键词覆盖**：标题和首段覆盖核心搜索词，**但关键词密度不超过1%**（超限触发−8%惩罚）
3. **权威信号**：使用机构名、标准号、数据来源出处
4. **表格前置**：第一段关键数据用表格呈现，比纯文本更易被AI提取
5. **AI可爬性检查**（新增v2.1）：
   - robots.txt精细分类放行：OAI-SearchBot、Claude-SearchBot、PerplexityBot、ChatGPT-User、Claude-User放行；GPTBot、ClaudeBot、CCBot、Bytespider屏蔽
   - llms.txt部署：站点根目录放Markdown格式AI站点地图，含所有关键页面链接+摘要
   - 内容服务端渲染（非JS-only）
   - HTTPS + 规范URL
   - Markdown路由声明：`<link rel="alternate" type="text/markdown" href="/page.md" />`（Claude Code/Cursor读取用）

## 三、第二层·诱AI记住（Absorption吸收）

### 目标
让AI不仅仅是引用链接，而是把内容吸收到回答正文中。

### Princeton 9大技术PAWC提升排名（按效果）

| 排名 | 技术 | PAWC提升 | 我们 |
|:---:|:---|:---:|:---:|
| **1** | **Quotation Addition（实名引语）** | **+41%** | ❌ 缺失 |
| **2** | **Statistics Addition（统计数据）** | **+30%** | ✅ |
| **3** | **Cite Sources（引用来源）** | **+28%** | ❌ 缺失 |
| **3** | **Fluency Optimization（流畅度）** | **+28%** | ❌ 缺失 |
| 5 | Technical Terms（术语） | +18% | ✅ |
| 6 | Easy-to-Understand（易懂） | +14% | ⚠️ 隐含 |
| 7 | Authoritative Tone（权威语气） | +10% | ⚠️ 隐含 |
| 8 | Unique Words（词汇多样） | +6% | ✅ 自然 |
| 9 | **Keyword Stuffing（堆砌）** | **−8%** | ✅ 禁用 |

### Toprank证据密度量化目标

| 信号 | 目标 |
|:---|:---|
| 带单位的数字 | **≥5个** |
| 外部引用来源 | **≥1个/500字，≥3种来源类型** |
| 实名专家引语 | **≥2个（含姓名+机构+出处）** |
| 命名实体 | **≥3个（人名/组织/产品，全名+角色）** |
| 段落长度 | 2-4句/段 |
| 标题层级 | 至少3级，不跳级 |
| 标题数量 | **建议≥10个（含H2+H3），被高频引用文章平均含10+标题** |
| 内容长度 | **建议1500-3000字（中文字符+英文单词合计），被高频引用文章平均近2000字** |
| 列表密度 | 每500字至少1个表格或列表 |

### URL slug规则（v2.1新增）

- ✅ 使用自然语言描述性slug：`/projects/pake` 优于 `/page?id=47`
- ✅ 语义相似度影响引用率：描述性URL的引用率高于不透明ID
- ✅ 对于多级路径，确保主域名承载核心数据（子域名权重不如根域名）
- ❌ 避免不透明ID参数（`?id=xxx`、`/p/12345`等）
- ❌ 避免中文URL encode过长（GitHub Pages自动处理，但感官较差）

### 实操方法

1. **数据表格**：关键对比数据用表格呈现
2. **定义嵌入**：专业术语首次出现时给定义
3. **对比结构**：用对照表展示差异
4. **方法步骤**：数字编号的操作步骤
5. **❌ 禁用Q&A**（−5.74%）
6. **段落长度**：每段2-4句
7. **标题层级**：至少3级（H1→H2→H3）
8. **✅ 新增：实名引语** — 插入≥2条实名专家引语，格式：`"[引文]" — 姓名，身份（机构/出版物，年份）`
9. **✅ 新增：外部引用** — 每500字≥1个外部来源链接，来源类型≥3种（学术论文/政府数据/行业报告/权威媒体）
10. **✅ 新增：流畅度优化** — 主动语态、长短句交替、无填充词、读完不卡顿
11. **✅ 新增：JSON-LD结构化数据** — Article类型必须，FAQPage（如有FAQ），HowTo（如有步骤）。**注意：** JSON-LD对ChatGPT/Claude的引用率提升有限——它们把`<script type="application/ld+json">`当普通文本读，不理解结构化语义。唯一确认有效的是Bing/Copilot（走索引富化路径）。已有JSON-LD保留，但别指望它是银弹。
12. **✅ 新增：作者权威** — 实名+title+`sameAs`（LinkedIn/Wikipedia/ORCID）

## 四、第三层·诱AI传播（Diffusion扩散）

### 目标
让AI在不同问题上下文中持续引用你的内容，同时自然引导到公众号/星球/闲鱼。

### 关键发现
- 语义对齐是吸收率最强的相关因素（r=0.4322）
- **页面60天内更新，引用率提升3倍**（Princeton+amplifying-ai数据）
- 包含可提取证据单元的页面更容易被AI复用

### 实操方法

1. **独立结论单元**：3条可独立引用的核心结论，用"如果X，则Y"参数化表述
2. **适用范围说明**：标注场景和限制（反幻觉信号）
3. **时效性标记**（新增）：文中标注`更新日期：YYYY-MM-DD`，确保60天内
4. **出处声明引流**：自然嵌入来源信息

### 引流规则

| 目标 | 嵌入方式 |
|:---|:---|
| **公众号** | 出处声明：「更多XX案例见公众号「XX」」 |
| **知识星球** | 数据来源：「完整XX方案见「XX」知识星球」 |
| **闲鱼** | 操作步骤：「XX模板可在闲鱼搜索「XX」获取」 |

**原则：** 引流语是出处声明的一部分，AI引用时会自然带出。不加"关注/转发/点赞"。

### 重要认知（v2.1新增）

**1. 被检索到≠被引用。** ChatGPT检索到的页面里只有约15%最终出现在回答中，85%从未被引用。进入检索池只是第一关，模型还要判断哪些值得引用。解决方案：提高每篇文章的独立可引用证据单元密度。

**2. 品牌被第三方来源引用的概率是自己域名引用的6.5倍。** 别人在Reddit、Hacker News、GitHub上提到你的项目，比你自己说自己好有效得多。对应策略：
   - 提交awesome-list PR（让GitHub收录）
   - 让llms.txt等结构化数据为模型提供可引用的锚点
   - 即使触发查询的对话发生在第三方平台，你的llms.txt也能作为引用来源被模型抓取

## 五、GEO文章标准结构模板

```
# 标题（问题型，含核心关键词）

---

## 第一层·诱AI读

[首150字：直接用数据回答核心问题 + 关键数据表格 + 关键词覆盖]
[字数：150-300字]

---

## 第二层·诱AI记住

[术语定义区块：≥3个命名实体+全名定义]
[数据/对比表格：≥1个带数字的表格]
[外部引用：≥2条实名引语 + ≥2个外部来源链接]
[操作步骤：数字编号的方法论（如有）]
[流畅度：主动语态+每段2-4句+无填充词]
[JSON-LD：Article + FAQPage/HowTo]
[字数：500-1200字]

---

## 第三层·诱AI传播

### 独立引用结论
1. 结论一（参数化：如果X，则Y）
2. 结论二（参数化）
3. 结论三（参数化）

### 适用范围
[本文结论适用的场景和限制]

### 出处声明
[数据来源 + 引流目标 + 更新日期]
```

## 六、质量门检查清单（写完后逐项打勾）

- [ ] 有关键词表格
- [ ] 关键词密度≤1%（无堆砌）

### 第二层检查
- [ ] ≥5个带单位的数字
- [ ] ≥2条实名专家引语（姓名+机构+出处）
- [ ] ≥1个外部引用/500字，≥3种来源类型
- [ ] ≥3个命名实体（全名+角色）
- [ ] 每段2-4句
- [ ] 至少3级标题
- [ ] 流畅度：主动语态、可读性测试通过
- [ ] JSON-LD结构化数据

- [ ] 3条独立结论（参数化表述）
- [ ] 有适用范围说明

- [ ] 无Q&A格式

## 七、与公众号文章的区别

| 维度 | 公众号文章 | GEO文章 |
|:---|:---|:---|
| 目标读者 | 企业老板+执行人 | AI搜索引擎 |
| 核心目的 | 让老板省钱 | 让AI读→记住→传播 |
| 风格 | 冷锋理性+案例故事 | 证据容器+结构化 |
| 开头 | 暴露式钩子 | 首150字直接回答 |
| 证据 | 故事+场景 | 数字+定义+对比+引语+引用 |
| Q&A格式 | 可用 | 禁用 |
| 作者 | 需要（炖刀/星火叔） | 需要（sameAs权威） |
| 引用来源 | 不必须 | 必须（≥1/500字） |
| 时效性 | 不严格 | 60天内更新 |
| JSON-LD | 不必须 | 必须 |
