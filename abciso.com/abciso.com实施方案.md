## 一、技术选型（到VPS直接装）

| 项 | 选择 | 原因 |
|----|------|------|
| 框架 | **Astro + Starlight** | 静态站、Markdown内容源、零JS、速度满分、SEO/GEO天然友好 |
| 内容管理 | 本地 Markdown → Git → Astro build | 复用已有 `/mnt/e/GEO文章/` 的文件，不用改格式 |
| 部署 | Caddy + 静态文件 | 自动HTTPS、单二进制、ARM兼容、配一行就行 |
| 新闻聚合 | Python脚本 cron每日跑 → 生成md → 触发rebuild | 轻量，不依赖数据库 |
| CDN | Cloudflare（可选） | 上线后加，不强依赖 |

**一句话**：Astro读md文件出站，Caddy挡前面，Python爬新闻。不用数据库，不装WordPress。

## 四、行业动态——新闻聚合脚本设计

**抓取源列表**（首批6个，后续可加）：

| 来源 | URL | 类型 | 更新频率 |
|------|-----|------|---------|
| ISO官网 | iso.org/news/ | RSS | 周更 |
| CNAS | cnas.org.cn | 网页抓取 | 日更 |
| CNCA（认监委） | cnca.gov.cn | 网页抓取 | 日更 |
| 工信部 | miit.gov.cn | 网页抓取 | 日更 |
| 国标委（SAC） | sac.gov.cn | 网页抓取 | 周更 |
| 国家能源局 | nea.gov.cn | 网页抓取 | 日更 |

**脚本流程**：

```
cron 每日 09:00

fetch_news.py:
  1. 遍历抓取源，爬取最新公告标题+链接+摘要
  2. 去重（标题相似度判断）
  3. 自动打标（匹配关键词 → 打上标准标签）
  4. 生成 /src/content/news/YYYY-MM-DD-slug.md
  5. 触发 Astro rebuild
```

**展示样式**：列表页如上，每条只显示标题+来源+日期，详情页显示摘要并链回原文。

## 七、VPS上线操作清单

VPS到手后，按这个顺序操作：

```
□ 1. 域名DNS指向VPS IP
□ 2. 安装 Caddy + Git + Python
□ 3. 从本地拉取 Astro 项目到 VPS
□ 4. 配置 Caddyfile 做静态文件服务和反向代理
□ 5. 首次 build → 确认网站可访问
□ 6. 配置 cron：每日 09:00 新闻聚合脚本
□ 7. 配置 cron：每日 09:30 重新 build（新闻刷新）
□ 8. 上传免费资料（检查表xlsx等）
□ 9. 提交 sitemap 到百度/Google
□ 10. 上线llms.txt → 确认AI可抓取
```
