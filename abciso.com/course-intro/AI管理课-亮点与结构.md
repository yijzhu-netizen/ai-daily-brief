# 《AI时代，你更应该学会管理》课程亮点与结构

***

## 课程定位

一门让普通人也能听懂的系统化管理课。

不讲理论史，不讲空洞概念。每讲一个工具，学了就能用。面向零基础——刚入职的管理培训生、技术转管理、创业者、对管理感兴趣但没学过的人。

对标罗宾斯《管理学》第13版教材框架，融合德鲁克、特劳特、稻盛和夫等经典著作，但用中国制造业的真实案例说话。

***

## 核心亮点

### 亮点一：AI时代差异化

市面上绝大多数管理课讲的还是20世纪的管理方法。这门课在每讲、每节、每个案例都扣住一个核心问题：**AI来了，管理是过时了，还是更重要了？**

结论清晰：AI能帮你做D（执行），但不能替代你的P（计划）和C（检查）。**管理能力越强的人，AI用得越顺手。**

### 亮点二：制造业真实案例

不用苹果谷歌海底捞丰田那些听烂的案例。全部来自作者十几年审核生涯的真实经历：

* **苏州上市公司**——美国抓马杜罗→预判原材料涨价→提前备库存锁低价

* **常州口罩公司**——疫情赚2000万→熔喷布全打水漂（把运气当能力的代价）

* **电装车间**——换模24小时→5分钟，十年持续改进的力量

* **皮带一裁为二**——6000→3000，成本省一半寿命翻一倍

* **检测室取料**——1.5小时降到40分钟，没花一分钱的流程创新

* **河南那家工厂**——大屏显示64%达成率，计划脱离资源的真实写照

每个案例有公司背景、有具体数据、有教训总结。

### 亮点三：不是权谋，是科学

专门用一整节讲清楚：管理可验证、可复制、可证伪。权谋靠的是拉帮结派、揣摩上意、秘而不宣。**越规范的组织，权谋越没用；越混乱的组织，权谋越吃香。你学管理，就是把你的组织从"靠关系"变成"靠规则"。**

### 亮点四：管P和C，不盯D

这是整门课的方法论内核。P（计划）决定了方向对不对，C（检查）决定了执行跑不跑偏。D（执行）让下属做、让团队做、让AI做。**盯D是救火队长，盯P和C才是管理者。**

### 亮点五：番外篇·真实AI系统拆解

Gate（审核助手）和Sage（创作研发助手）是作者每天在用的真实AI系统——不是靠代码写的，是靠课程里的管理方法论搭出来的。番外篇完整拆解了四套方法论在AI系统上的落地：做减法（从12个agent砍到8个）、一次只做一件事、资源梳理（不同模型就是不同的"机"）、用数据说话。

**"我不是程序员，不懂技术，但懂思路。我能搭出来，你也能。"**

## 课程结构

### 总览：16讲 + 番外篇 ≈ 7000行

```
┌─────────────────────────────────────────────────────────┐
│ 模块一：入门——管理是什么（2讲）                         │
│   第1讲 管理不是什么神秘的东西                          │
│   第2讲 管理的来龙去脉和中国现实                        │
├─────────────────────────────────────────────────────────┤
│ 模块二：P（计划）——定方向做决策（3讲）                  │
│   第3讲 计划——不会计划就是在计划失败                    │
│   第4讲 决策——天天在做但经常做错的事                    │
│   第5讲 变革与创新——不变就是等死                        │
├─────────────────────────────────────────────────────────┤
│ 模块三：D（执行）——搭班子带队伍（8讲）                  │
│   第6讲 组织结构——别把公司搭成违章建筑                  │
│   第7讲 人力资源管理——人不是成本是投资                  │
│   第8讲 团队——一个人走得快，一群人走得远                │
│   第9讲 战略管理——站在高处看全局                        │
│   第10讲 领导不是职位，是影响力                        │
│   第11讲 激励——别人凭什么好好干                         │
│   第12讲 沟通——管理就是说话                            │
│   第13讲 情绪智力——管理者最被低估的能力                │
├─────────────────────────────────────────────────────────┤
│ 模块四：C（控制）——管过程看结果（1讲）                  │
│   第14讲 控制——不是束缚，是确保方向                    │
├─────────────────────────────────────────────────────────┤
│ 模块五：A（改进+收口）（2讲）                            │
│   第15讲 运营管理与质量——持续改进的底层逻辑            │
│   第16讲 做个真正的管理者——五条原则+PDCA地图           │
├─────────────────────────────────────────────────────────┤
│ 番外篇：用管理的方法用AI——Gate与Sage实战拆解            │
└─────────────────────────────────────────────────────────┘
```

### 每讲结构（8环节）

```
【环节0】自我介绍 → 每讲开头一字不差，建立人设
【环节1】一句话开场 → 反常识问题/具体场景，"你遇到过吗"
【环节2】本讲学什么 → 3-5个"你能……"目标，至少一条AI时代相关
【环节3】核心内容 → 理论+大白话+箭头案例+AI时代的启示
【环节4】实操工具 → 一张表/一个模板，学员下课就能用
【环节5】经典观点 → 德鲁克/稻盛/特劳特原文引用+一句话翻译
【环节6】本讲小结 → 数字编号，以"所以AI时代你更应该学会管理"收口
【环节7】课后作业 → 一个具体动作，含AI时代引导
```

### 贯穿全课的几条线

| 线索             | 起点               | 展开                    | 收口                 |
| :------------- | :--------------- | :-------------------- | :----------------- |
| **AI时代管理视角**   | 第1讲开场即AI钩子       | 每讲每节自然嵌入              | 第16讲"AI是油门，管理是方向盘" |
| **PDCA底层系统**   | 第1讲埋种子（Hermes案例） | 第14讲正式展开              | 第16讲PDCA地图完整收口     |
| **管理≠权谋**      | 第1讲独立章节          | 第2讲人情管理呼应             | 第16讲原则二（信任专业的人）    |
| **管P和C，不盯D**   | 第1讲（管理四职能）       | 每讲扣落脚点                | 第16讲"你管P和C，AI管D"   |
| **提示词=给AI的计划** | 第3讲（SMART套提示词）   | 第12讲（沟通呼应）→第14讲（控制呼应） | 番外篇全流程拆解           |

## 适用场景

* **企业内训**：面向技术转管理、新晋管理者、程序员等带团队的职场人

* **知识付费**：知识星球/付费专栏，定价299元（AI时代×管理无直接竞品）

* **自学入门**：完全不懂管理的人，学完能理解管理是什么、管理者干什么、怎么管人管事

## 课程时长

每讲阅读约15-20分钟，总计约5-6小时。
可按模块分段学习，每讲独立，不依赖前后顺序。
